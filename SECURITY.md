# Security Policy

## Reporting a Vulnerability

**Do not report security vulnerabilities through public GitHub issues.**

Please report them privately via email to `<security@yourproject.com>`.

In your report, please include:
*   A detailed description of the vulnerability.
*   Steps to reproduce the issue.
*   The potential impact.
*   Your preferred contact method for follow-up.

We aim to acknowledge your report within 48 hours.

## Supported Versions

Security updates are provided for the following supported versions:
*   `1.x` - Actively supported
*   `0.x` - No longer supported

We recommend upgrading to the latest stable version to receive all security fixes.

## Disclosure Policy

We follow a Coordinated Vulnerability Disclosure (CVD) process. After receiving a report:
1.  We will confirm and triage the issue.
2.  We will work on a fix and keep the reporter informed.
3.  Once a patch is ready, we will release an update.
4.  We will then publicly disclose the vulnerability details.
